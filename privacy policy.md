# **LzveRed Discord Bot – Privacy Policy**

**Last Updated:** April 12, 2026

---

## 1. Overview

LzveRed (“the Bot”) is a multi-purpose Discord application providing moderation, logging, automation, and server management functionality.

By inviting or using LzveRed, you (“User” or “Server Member”) agree to the data practices outlined in this Privacy Policy.

---

## 2. Data Controller & Responsibility

LzveRed is operated and maintained by a single controlling entity (“Bot Owner”).

The Bot Owner is responsible for:

* Data storage and processing decisions
* System configuration and moderation logic
* Enforcement of deletion and retention policies
* Administrative override functions

There is no decentralized ownership or shared administrative authority outside the Bot Owner.

---

## 2.5 Administrative Authority Hierarchy

LzveRed operates under a strict tiered access model:

### **1. Bot Owner (System Controller)**

The Bot Owner has full and unrestricted control over all bot systems, including:

* Access to all stored data
* Modification or deletion of any record
* Execution of owner-level management commands (including `ownermanagement`)
* Configuration of retention, logging, and deletion systems
* Override authority over all server-level settings and user requests

### **2. Server Administrators**

Server administrators may:

* Configure bot features within their server
* Enable or disable bot functionality where permitted
* Manage moderation tools provided by the Bot

Server administrators do **not** have access to global data systems or owner-level commands.

### **3. End Users**

End users may:

* Interact with bot features within server permissions
* Request deletion of their data (`,mydata forgetme`)

End users do not have access to administrative or owner-level systems.

---

## 3. Data We Collect

LzveRed collects and processes the following categories of data:

### 3.1 User Data

* Discord User IDs
* Discord usernames
* Messages sent in servers where the Bot is active

### 3.2 Server Data

* Discord server (guild) IDs
* Server names
* Configuration and moderation settings

### 3.3 Activity & Moderation Data

* Command usage logs
* Anti-spam and automation detection logs
* Moderation actions (warnings, kicks, bans)
* Audit and compliance records

---

## 4. How Data Is Used

Collected data is used strictly for:

* Moderation enforcement (automated and manual systems)
* Anti-spam and abuse prevention
* Server logging and audit functionality
* Feature operation and system reliability
* Compliance and security monitoring

---

## 5. Data Storage & Retention

* All data is stored locally under systems controlled by the Bot Owner.
* Data is retained until:

  * The user leaves a server, or
  * The data is manually deleted by the Bot Owner, or
  * A deletion request is processed under enabled policies

Retention periods may vary depending on system configuration, operational requirements, and security needs.

---

## 6. Data Deletion & Management System

LzveRed provides a structured multi-level data management system.

### 6.1 User-Initiated Deletion

Where enabled by server configuration, users may request deletion of their data.

### 6.2 Deletion Command System

Users may initiate deletion requests via:

* `,mydata forgetme`

This request triggers internal processing that removes user-associated data where technically feasible.

### 6.3 Owner-Level Deletion

The Bot Owner may:

* Delete any user data at any time
* Modify or override deletion states
* Execute global or targeted data removal using administrative tools (including `ownermanagement`)

### 6.4 Discord Compliance Requests

The Bot supports processing external or Discord-initiated data removal requests through internal handling systems.

### 6.5 Scope of Deletion

Depending on request type and configuration, deletion may include:

* User IDs
* Usernames
* Message logs
* Activity records
* Moderation-related metadata

### 6.6 Limitations

* Some data may persist temporarily in backups until overwritten or purged
* Some logs may be retained for security, abuse prevention, or compliance
* Server-wide logs may not always be attributable to individual users

---

## 7. Data Sharing

LzveRed:

* Does not sell user data
* Does not share data with third-party services
* Does not distribute stored data externally unless legally required

All data remains under direct control of the Bot Owner.

## 8. Permissions & Access Requirements

LzveRed requires elevated Discord permissions, including administrator-level access in many servers.

By granting permissions, server owners acknowledge:

* The Bot may read messages in channels it has access to
* The Bot may execute moderation actions
* The Bot may log activity within its operational scope

## 9. Automated Processing

LzveRed uses automated systems to:

* Detect spam and abusive behavior
* Trigger moderation actions (warnings, kicks, bans)
* Log server and user activity

These systems operate based on predefined rules and thresholds configured by the Bot Owner or server settings.
